# GitHub Pages 대시보드 배포 가이드

## 1) 파일 구성
- `index.html` : 화면 레이아웃
- `styles.css` : 화면 스타일
- `script.js` : 데이터 로드, KPI 계산, 차트 렌더링, 화면 잠금
- `data.json` : 엑셀에서 정리한 매출 데이터

## 2) 바로 올리는 방법
1. GitHub에서 새 저장소를 만듭니다.
2. 위 4개 파일을 저장소 루트에 업로드합니다.
3. 저장소 `Settings` → `Pages`로 이동합니다.
4. `Build and deployment`에서 `Deploy from a branch`를 선택합니다.
5. Branch는 `main`, Folder는 `/ (root)`를 선택 후 저장합니다.
6. 잠시 후 `https://사용자명.github.io/저장소명/` 형태로 접속됩니다.

GitHub Pages는 저장소의 정적 파일을 그대로 배포할 수 있고, 배포 소스로 특정 브랜치나 `/docs` 폴더를 지정할 수 있습니다. 또한 공개 저장소뿐 아니라 일부 플랜에서는 비공개 저장소에서도 사용할 수 있습니다. citeturn755037search5turn755037search7

## 3) 비밀번호 예시 구조
현재 예시는 `script.js` 안의 SHA-256 해시와 사용자가 입력한 비밀번호 해시를 비교하는 방식입니다.
- 예시 비밀번호: `bomwoo2026`
- 이 방식은 **화면 잠금용**일 뿐입니다.
- 개발자도구를 열면 코드와 데이터가 보입니다.

GitHub Pages는 정적 사이트 호스팅이므로 서버 측 인증을 직접 처리하지 않습니다. 공개된 Pages 사이트는 인터넷에서 접근 가능하므로 민감한 데이터를 저장소에 넣으면 안 됩니다. citeturn755037search6turn755037search0

## 4) 실제 보안이 필요할 때
다음 중 하나를 권장합니다.
- Cloudflare Access를 붙여 이메일/OTP 인증 적용
- 사내 VPN 안에서만 접속 가능한 내부 서버 사용
- 별도 백엔드(예: Azure, AWS, 사내 IIS/Nginx) + 로그인 세션 구성
- GitHub는 코드 보관용으로만 사용하고 실제 데이터는 API/DB에서 호출

## 5) 엑셀 → JSON 갱신 방법
운영 구조는 아래처럼 가져가면 가장 편합니다.
1. 월별 원본 엑셀 업데이트
2. 정리데이터 시트 기준으로 `Date / Section / Entity / Sales(억) / 3M_MA(억)` 구조 유지
3. 같은 형식으로 `data.json` 재생성
4. GitHub에 파일 교체 후 커밋
5. Pages가 자동 반영

## 6) 추천 운영 방식
- 공개 데모: 현재 구조 그대로 가능
- 실데이터 운영: 민감한 거래처명/원가/개별단가는 제거
- 경영 보고: KPI 카드 + 월별 추이 + 최신월 비교 + 섹션 구성 정도로 단순화 유지

## 7) GitHub Pages 한계
GitHub Pages는 정적 사이트 배포에 적합하지만 용량, 빌드 시간, 대역폭 등에 제한이 있습니다. 예를 들어 게시된 사이트 크기는 1GB 이하, 배포 시간 제한은 10분, 월간 대역폭은 소프트 리밋 100GB입니다. citeturn755037search2
